# A11y Quick Scan (2026-06-15)

## Methodology

Light scan of 4 key user-facing pages (home, product, compare, search) + 2 form pages (login, register) + UI primitives (button, input).

## Strengths ✅

1. **Base UI primitives** (`@base-ui/react`) — accessibility built-in (focus rings, ARIA attrs, keyboard nav)
2. **SkipToContent** link in root layout — keyboard users can jump past nav
3. **Form labels** — `LoginForm`, `RegisterForm`, `PriceAlertForm`, `ReviewForm` all use `<Label htmlFor>` pattern
4. **Semantic HTML** — pages use `<h1>`, `<h2>`, `<nav>`, `<main>` correctly
5. **404 / not-found** uses semantic structure with heading hierarchy
6. **No missing alt text** in core components (verified by grep)

## Gaps Found & Fixed

### A11y-1: Search input missing `aria-label` ✅ FIXED
- **File:** `src/components/search/SearchBar.tsx:55`
- **Issue:** No `aria-label`, no `<label>`. Placeholder-only is invalid a11y.
- **Fix:** Added `aria-label="Cari produk"`
- **Impact:** Screen readers can now announce input purpose.

### A11y-5: Icon-only buttons missing labels ✅ FIXED (2 of N)
- **File 1:** `src/components/search/SearchBar.tsx:67-77` — clear search button
- **File 2:** `src/components/admin/offers-list.tsx:128` — refresh offers button
- **Issue:** Icon-only `<Button size="icon">` with no aria-label
- **Fix:** Added `aria-label="Hapus pencarian"` and `aria-label="Refresh offers list"`
- **Impact:** Screen reader users can understand button purpose
- **Note:** Other icon buttons may need similar treatment — mechanical sweep recommended (Phase 3)

## Remaining Gaps (Phase 3 backlog)

### A11y-2: No live region for search results
- **File:** Search results UI
- **Issue:** New search results don't announce to screen readers
- **Fix:** Add `aria-live="polite"` to results container
- **Effort:** Small

### A11y-3: Color contrast for `text-blue-100` on `bg-blue-600`
- **File:** Home page header
- **Issue:** Subtitle text may not meet WCAG AA contrast
- **Fix:** Darken text or add shadow
- **Effort:** Small

### A11y-4: No reduced-motion support
- **Files:** Any CSS animations
- **Issue:** Users with `prefers-reduced-motion` not respected
- **Fix:** Add `motion-reduce:` Tailwind variants or `@media (prefers-reduced-motion)`
- **Effort:** Medium

### A11y-5: Icon-only buttons missing labels
- **Files:** Many (admin offers-list, etc.)
- **Issue:** Icon-only `<Button>` without `aria-label`
- **Fix:** Audit + add aria-label (2 of N done — SearchBar, offers-list)
- **Effort:** Small (mechanical sweep)

## Verdict

**A11y baseline is solid** (Base UI primitives + form labels + skip link). 1 gap fixed today. 4 gaps remain — all low-priority improvements, not blockers.

## Out of Scope

- Full WCAG 2.1 AA audit (needs axe-core or Lighthouse audit)
- Keyboard navigation testing (manual QA needed)
- Screen reader testing (NVDA/JAWS)
- Color contrast audit (needs automated tool)
