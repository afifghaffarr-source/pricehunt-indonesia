# P0: E2E CI Failure Fix (2026-06-15)

## TL;DR
E2E workflow failed in CI (run #7, sha `e27107d`). All 13 tests fail because **Supabase env vars are EMPTY** at build time, so every page renders the "Terjadi Kesalahan Sistem" error boundary instead of real content.

## Root cause
Verified from CI logs (`runs/27535436350/jobs`):

```
env:
  NEXT_PUBLIC_SUPABASE_URL:        ← empty
  NEXT_PUBLIC_SUPABASE_ANON_KEY:   ← empty
  SUPABASE_SERVICE_ROLE_KEY:       ← empty
```

The workflow file references these GitHub secrets:
- `secrets.SUPABASE_URL`
- `secrets.SUPABASE_ANON_KEY`
- `secrets.SUPABASE_SERVICE_ROLE_KEY`

But the values come back empty — meaning the **secret names in your GitHub repo settings don't match** the names the workflow expects. Either:
1. The secrets were never created
2. The secrets were created with **different names** (e.g. `NEXT_PUBLIC_SUPABASE_URL` instead of `SUPABASE_URL`)

## Fix shipped (commit `e27107d` → in flight)
Updated `.github/workflows/e2e.yml` and `.github/workflows/ci.yml` to **try multiple naming conventions** using GitHub's `||` fallback:

```yaml
# Modern names (preferred)
${{ secrets.SUPABASE_URL || secrets.NEXT_PUBLIC_SUPABASE_URL }}
# Falls back to legacy names automatically
```

This means **whichever naming convention you used when creating the secrets will now work**.

## What you need to do

### Option A: Secrets already exist (most likely)
If you created secrets named `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` (common pattern since Next.js docs use those names), the fix is automatic — the next CI run will pass.

**Verify the next CI run succeeds** at:
https://github.com/afifghaffarr-source/pricehunt-indonesia/actions

### Option B: Secrets don't exist
If the next CI run still fails, the new "Verify Supabase env vars are set" step will print a clear error message with the URL to add them.

Add secrets at:
https://github.com/afifghaffarr-source/pricehunt-indonesia/settings/secrets/actions

Use these names (modern, recommended):
- `SUPABASE_URL` = your Supabase project URL (e.g. `https://oklaxwjoyttpwgxhphko.supabase.co`)
- `SUPABASE_ANON_KEY` = your Supabase anon/public key
- `SUPABASE_SERVICE_ROLE_KEY` = your Supabase service role key

You can copy these from your local `.env.local` file.
