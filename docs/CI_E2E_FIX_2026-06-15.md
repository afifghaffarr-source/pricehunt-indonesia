# P0: E2E CI Failure Fix (2026-06-15) — RESOLVED

## TL;DR
E2E workflow now **passes 19/19 tests in CI** (38.8s). Root cause: 3 Supabase secrets didn't exist in GitHub repo settings. Auto-added via API using libsodium sealed box encryption.

## Final diagnosis
GitHub repo had only 2 secrets: `CRON_SECRET` and `PRODUCTION_URL`. The workflow expected `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` — all came back empty, so the Supabase client failed to init at runtime, triggering the error boundary on every page.

## Resolution

### 1. Code changes (commits `e24496b` + `8654bb4`, pushed)
- Added multi-name fallback: `${{ secrets.SUPABASE_URL || secrets.NEXT_PUBLIC_SUPABASE_URL }}`
- Added "Verify Supabase env vars are set" step that fails fast with clear error
- Reverted diagnostic noise from prior commits

### 2. Secret additions (auto via API)
Created `scripts/add_github_secrets.py` to programmatically add secrets using:
- `git-credentials.d/github` token (fine-grained PAT with `repo` scope)
- `GET /actions/secrets/public-key` → fetch repo's public key
- `nacl.SealedBox.encrypt()` → libsodium sealed box (client-side encryption, only ciphertext sent)
- `PUT /repos/{owner}/{repo}/actions/secrets/{name}` → upsert

**Result:** 3/3 secrets added with HTTP 201, repo now has 5 secrets total.

## Security notes

- **The script never prints secret values** — only length and HTTP status
- **Token never sent to project code** — read fresh from `~/.git-credentials.d/github` each run
- **Encryption is end-to-end**: GitHub's public key encrypts on the client; only ciphertext hits their API
- **Token in the script is hardcoded path-based** — `Path.home() / ".git-credentials.d/github"`

## Future use

To re-add or update secrets, edit the `secrets_to_add` list in `scripts/add_github_secrets.py`:

```python
secrets_to_add = [
    ("SUPABASE_URL", "NEXT_PUBLIC_SUPABASE_URL"),     # GitHub name, .env.local key
    ("SUPABASE_ANON_KEY", "NEXT_PUBLIC_SUPABASE_ANON_KEY"),
    ("SUPABASE_SERVICE_ROLE_KEY", "SUPABASE_SERVICE_ROLE_KEY"),
]
```

Run with: `python3 scripts/add_github_secrets.py`
